Ошибки (исключения) — неизбежная часть программирования. Python предоставляет мощные инструменты для их обработки и отладки кода.

🐞
Типы ошибок
  • Синтаксические ошибки — возникают, когда код не соответствует правилам языка (например, забыто двоеточие).
  • Исключения (ошибки времени выполнения) — возникают во время выполнения программы при корректном синтаксисе (например, деление на ноль).
🛡️
Обработка исключений: try / except
try:
    number = int(input("Введите число: "))
    result = 10 / number
    print("Результат:", result)
except ZeroDivisionError:
    print("Ошибка: деление на ноль!")
except ValueError:
    print("Ошибка: введено не число")

Блок try содержит код, который может вызвать исключение. Если исключение возникает, выполнение переходит в соответствующий блок except.

⚙️
Блоки else и finally

else выполняется, если исключений не возникло.
finally выполняется всегда, независимо от того, было исключение или нет (например, для закрытия файлов).

try:
    f = open("file.txt")
    data = f.read()
except FileNotFoundError:
    print("Файл не найден")
else:
    print("Файл прочитан успешно")
finally:
    f.close()  # выполнится в любом случае
🚨
Поднятие исключений с raise

Можно самостоятельно вызвать исключение с помощью raise.

def withdraw(amount, balance):
    if amount > balance:
        raise ValueError("Недостаточно средств")
    return balance - amount
✅
Отладка с assert

assert проверяет условие и, если оно ложно, вызывает исключение AssertionError. Удобно для отладки.

def divide(a, b):
    assert b != 0, "Делитель не может быть нулём"
    return a / b

Можно отключить все assert, запустив Python с ключом -O.

🖨️
Отладка с помощью print

Самый простой способ отладки — выводить значения переменных в ключевых точках.

def factorial(n):
    print(f"Вызов factorial с n={n}")
    if n == 0:
        return 1
    result = n * factorial(n-1)
    print(f"Возврат {result} для n={n}")
    return result
