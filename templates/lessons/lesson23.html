В Python переменные хранят не сам список, а ссылку на него. Это важное отличие влияет на то, как списки ведут себя при присваивании, копировании и передаче в функции.

🔗
Переменная — это ссылка

Когда мы создаём список, Python выделяет память для хранения его элементов. Переменная лишь хранит адрес этой области памяти (ссылку).

a = [1, 2, 3]  # a ссылается на список
b = a           # b ссылается на тот же список

Переменные a и b указывают на один и тот же объект в памяти.

🔄
Изменения видны через все ссылки
a = [1, 2, 3]
b = a
b.append(4)
print(a)  # [1, 2, 3, 4]  — изменилось и в a!

Это происходит потому, что a и b указывают на один объект. Модификация через любую переменную меняет сам список.

🔍
Оператор is и id()

Можно проверить, ссылаются ли переменные на один объект, с помощью is или id().

a = [1, 2, 3]
b = a
print(a is b)   # True
print(id(a) == id(b))  # True (адреса совпадают)

c = [1, 2, 3]
print(a is c)   # False — разные объекты
📋
Как создать независимую копию

Чтобы создать новый список с теми же элементами, используют:

  • Срез [:]
  • Функцию list()
  • Метод .copy()
a = [1, 2, 3]
b = a[:]      # копия через срез
c = list(a)   # копия через list()
d = a.copy()  # копия через метод

b.append(4)
print(a)  # [1, 2, 3] — оригинал не изменился
⚠️
Поверхностное копирование

Срез, list() и .copy() создают поверхностную копию. Если список содержит изменяемые объекты (например, вложенные списки), копируются только ссылки на них.

a = [[1, 2], [3, 4]]
b = a.copy()
b[0].append(99)
print(a)  # [[1, 2, 99], [3, 4]] — внутренний список изменился!

Для полной независимости нужно глубокое копирование (copy.deepcopy).

📦
Списки в функциях

При передаче списка в функцию передаётся ссылка, поэтому функция может изменить оригинал.

def add_element(lst, value):
    lst.append(value)

my_list = [1, 2]
add_element(my_list, 3)
print(my_list)  # [1, 2, 3]

Чтобы защитить список от изменений, можно передать копию.
