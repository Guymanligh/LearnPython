Функции и переменные тесно взаимодействуют. Понимание области видимости переменных и способов передачи данных в функции — ключ к написанию надёжного кода.

🔒
Локальные переменные

Переменные, созданные внутри функции, называются локальными. Они существуют только во время выполнения функции и недоступны снаружи.

def my_func():
    x = 10          # локальная переменная
    print(x)

my_func()           # 10
# print(x)          # Ошибка: name 'x' is not defined
🌍
Глобальные переменные

Переменные, объявленные вне функций, называются глобальными. Они доступны в любой части модуля, но внутри функции их нельзя изменить без специального ключевого слова.

y = 20              # глобальная переменная

def show():
    print(y)        # можно прочитать

show()              # 20
✏️
Изменение глобальных переменных

Чтобы изменить глобальную переменную внутри функции, нужно объявить её как global.

counter = 0

def increment():
    global counter
    counter += 1

increment()
increment()
print(counter)      # 2

Без global Python создаст локальную переменную с тем же именем.

📦
Передача изменяемых объектов

Если передать в функцию изменяемый объект (список, словарь), функция может его модифицировать, и изменения отразятся на оригинале.

def add_item(lst, item):
    lst.append(item)

my_list = [1, 2]
add_item(my_list, 3)
print(my_list)      # [1, 2, 3]

Неизменяемые объекты (числа, строки, кортежи) нельзя изменить — функция получит копию значения.

📤
Возврат значений

Функция может вернуть результат через return, и этот результат можно присвоить переменной.

def square(x):
    return x * x

a = square(5)
print(a)            # 25

Если функция ничего не возвращает, она возвращает None.

🔢
Пример: счётчик вызовов

С помощью глобальной переменной можно подсчитать, сколько раз вызывалась функция.

calls = 0

def greet(name):
    global calls
    calls += 1
    print(f"Привет, {name}!")

greet("Анна")
greet("Борис")
print(f"Функция вызвана {calls} раз(а)")  # 2
